01Who Am I?

I'm an AI operations assistant that works alongside the Delta team. I can read files, write reports, manage spreadsheets, search the web, look up information, and automate repetitive tasks. I'm available 24/7 via Telegram.

I am NOT a chatbot.

I have memory, workspaces, scheduled tasks, and reusable skills. Think of me as a team member who can be trained once and then repeat the work forever — but only if you set it up right.

02How I Think

🧠 Memory & Context

I wake up fresh every session. My memory comes from files in your workspace — I read them at the start of each conversation.

  • Good context = I understand your task immediately
  • No context = I'm guessing, and I might guess wrong
  • Outdated context = I'll do things the old way

This is why keeping your workspace context updated is the single most important thing you can do.

⚡ Downtime

⚠️ I can't read messages sent while I was offline

If I didn't respond, please resend. I catch up on scheduled tasks automatically, but I can't recover missed chat messages.

04Set Up Your Workspace

Before you start chatting, set up a proper workspace. This is the foundation for everything.

📌 Step 1: Create a Topic

In the Delta-HQ Telegram group, each area of work has its own Topic (thread). This keeps conversations organized and gives me the right context for each job.

💡 One topic = One domain

Examples: "logistics-tracking", "driver-management", "monthly-reports". Don't mix unrelated work in one topic.

📂 Step 2: Link to a Workspace

Each topic should be connected to a workspace folder with context files:

workspace/
├── your-project/
│   ├── CONTEXT.md        ← What this job is about, rules, key info
│   ├── references/       ← Templates, data files, examples
│   ├── logs/             ← Output logs, status tracking
│   └── SKILL.md          ← The reusable skill (see Section 6)

When I enter your topic, I read the context files first. Better context = better results from the very first message.

🔄 Step 3: Keep Context Updated

This is the step most people skip — and it's the most important one:

After every successful task, update your workspace context.

Did the rules change? Did you learn something new? Did the process improve? Write it down. Future-you (and future-me) will thank you.

WhenWhat to update
Task completed successfullyAdd notes on what worked, any exceptions found
I made a mistakeAdd the correct rule to CONTEXT.md so I don't repeat it
Process changedUpdate the steps, remove outdated info
New data sourceAdd file IDs, sheet names, column mappings

Think of it like training a new employee: the better your training manual, the less you need to repeat yourself.

05The Right Workflow

Most people start by chatting with me like a search engine. That works for quick questions. But for real work, follow this flow:

🔄 The Delta AI Workflow

Phase 1: Define → Document

1. NAME THE JOB
   "This task is called: Monthly Cost Report"

2. WRITE THE DOCUMENT
   - What is the goal?
   - What are the inputs? (files, sheets, data sources)
   - What are the rules? (calculations, formats, exceptions)
   - What is the output? (where does it go, who gets it?)
   - What does "done correctly" look like?

3. SAVE TO WORKSPACE
   → your-project/CONTEXT.md

Phase 2: Chat → Refine

4. WORK WITH ME IN CHAT
   - Give me the document, test the process
   - I'll make mistakes — correct me
   - Each correction improves the document

5. UPDATE CONTEXT
   - After each successful run, update CONTEXT.md
   - Add edge cases, exceptions, lessons learned

Phase 3: Build → Automate

6. BUILD A SKILL
   - Package the process into a reusable skill (see Section 6)
   - Skill = documented process + scripts + references

7. SET A SCHEDULE (optional)
   - Schedule as a cron job: "Run every Monday at 8AM"
   - Now it runs automatically without chatting

8. REUSE & IMPROVE
   - Same skill, different data, forever
   - Update the skill when rules change
❌ What NOT to do

Don't just chat the same instructions every time. That wastes your time AND overloads me with context I have to re-process. Build a skill once, reuse forever.

06Build Skills, Not Chat

This is the key insight that separates productive AI use from just chatting:

❌ Chat approach
  • Explain the same task every time
  • I re-read context from scratch
  • Inconsistent results
  • Can't be automated
  • Knowledge lives in chat history (lost)
✅ Skill approach
  • Define once, run forever
  • I know exactly what to do
  • Consistent, repeatable results
  • Can run on schedule (cron)
  • Knowledge lives in files (permanent)

What is a Skill?

A skill is a packaged, reusable procedure with:

  • 📄 SKILL.md — Step-by-step instructions I follow
  • 📁 references/ — Templates, rules, mappings
  • 📜 scripts/ — Automation code (if needed)
  • 📋 CONTEXT.md — Domain knowledge and edge cases

Skill lifecycle

CHAT (explore & learn)
  ↓
DOCUMENT (write rules & process)
  ↓
BUILD SKILL (package into reusable format)
  ↓
TEST (run 2-3 times, fix issues)
  ↓
SCHEDULE (set cron job if recurring)
  ↓
MAINTAIN (update when rules change)

💡 Ask me to build the skill for you

After we've successfully done a task 2-3 times through chat, say: "Package this as a skill". I'll create the SKILL.md, reference files, and scripts automatically.

07Writing Better Prompts

🎯 The S.C.R. Formula

Specific → Context → Result

Tell me what to do, where to do it, and what good looks like.

Bad → Good examples

❌ Vague✅ Clear
"Check the report" "Open March report on Sheets, check if column F totals match column H"
"Send to team" "Send this summary to Delta-HQ, logistics topic"
"Fix numbers" "Row 25, change amount from 15,000 to 18,500 (price updated)"
"Remember this" "Remember this rule and save to the project context: orders over 500kg need separate invoice"

📋 Multi-step: use numbered lists

Do these 3 things:
1. Download today's tracking file
2. Check for arrivals tomorrow
3. Post summary to group chat

📎 Sending files: always add instructions

  • ✅ "Here's the file. Process it into an Excel report."
  • ❌ Dropping a file with no context (I'll guess — might guess wrong)

🔄 Iterating: be specific about changes

  • ✅ "Good, but change date to DD/MM/YYYY and add totals"
  • ❌ "Not quite, try again" (try what differently?)

08Teaching Me & Updating Context

The 4-step teaching method

1. STATE the rule
   "All international orders need a certificate number"

2. GIVE examples (right AND wrong)
   ✅ Order #123 with cert "VN-2026-456" → correct
   ❌ Order #124 with no cert → wrong

3. SAVE IT
   "Add this to the project context"

4. TEST ME
   "New order, no cert. What should I do?"

🔄 The context update habit

Every time something changes, close the loop:

✅ After success

"That worked. Update the context: the new rate is 5.2% effective April 1."

🔧 After fixing an error

"That was wrong. Save this rule: X always means Y, never Z."

📝 After process change

"We changed the workflow. Update the skill: step 3 is now before step 2."

🗑️ Removing outdated info

"Remove the old vendor list from context. We only use the new one now."

⚠️ If you don't update context, I'll use outdated rules

I only know what's in my files. If the rules changed but the files didn't, I'll keep doing it the old way — confidently and consistently wrong.

09Trust & Safety

✅ I will
  • Work accurately and consistently
  • Keep your information private
  • Ask before sending externally
  • Tell you when unsure
  • Log actions for transparency
❌ I won't
  • Share data outside approved channels
  • Send messages without permission
  • Delete files without asking
  • Make up information
  • Pretend to know what I don't

Trust levels

TRUST Skill-based automated tasks that have been tested
SPOT-CHECK Familiar task with new data — verify a few rows
VERIFY First-time task or new rule — always check the first run

10Quick Reference

🚀 Getting started checklist

  1. Create a Topic in Delta-HQ for your work area
  2. Set up a workspace folder with CONTEXT.md
  3. Start with chat to explore and refine the process
  4. After 2-3 successful runs, build a skill
  5. Set a cron schedule if it's recurring
  6. Update context whenever rules change

✨ Useful phrases

Say thisI'll do this
"This task is called: [name]"Create a named workspace for it
"Save this to context"Update the workspace CONTEXT.md
"Package this as a skill"Create SKILL.md + references + scripts
"Run this every [schedule]"Set up a cron job using the skill
"What do you know about [X]?"Show what's in my context files
"That's wrong because..."Learn the correction and update context
"Simpler please"Give a shorter answer

🔧 When things go wrong

  1. Wrong answer → Tell me what's wrong, why, and say "save this rule"
  2. Not responding → Probably offline. Resend after 30 min.
  3. Keeps repeating error → Context might be outdated. Say "update context: [correct info]"
  4. Overloaded / slow → Break into smaller tasks, or build a skill so I don't re-process everything
💡 The golden rule

Chat to explore. Document to remember. Build skills to scale.
The less you repeat yourself, the more I can do for you.
